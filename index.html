Jmeter

Jmeter5

采样器

基于协议： HTTP, FTP, HTTPS 等

HTTP Request： 常用HTTPCLient4
常用设置： jmeter.properties
Workbench 中的mirror server 可以帮助调试HTTP 请求
去请求文件下载，使用的是POST请求。如果需要保存server所推过来的文件/response保存下来，则可以添加一个listner 去把文件存储下来： Save Response to a file， 下载之后jmeter 会将其保存在jmeter 的bin 目录下。

如果想要通过jmeter上传文件，则也可以upload， 但一般情况下，jmeter 上传文件后消耗会到网络端，而不会很好的测试到server 的性能。

默认解析器： htmlparser
通过htmlparser.classname
通过设置Retrive All Embedded Resources from HTML Files 和 Use Concurrent Pool 更真实的模拟负载

URLS must match 可以帮忙过滤某些特定的url

IP 欺骗> Source Address
作为IP欺骗的地址，所使用的IP 地址不能是动态的，必须是静态的。

File协议
可以读取到本地的文件，并读取出来。
加入后置处理器后，就可以进行使用。

参数化

那些需要参数化

登陆认证信息
一些和时间相关的，违反时间约束的
一些受其他字段约束的
一些来自于其他数据源（如数据库的）
其他在系统运行过程中需要变动的

变量是每个线程都有各自一份的，一个线程修改并不影响另一个线程。

实现方式：
-通过CSV Data Set Config 组件
-User Defined Variables
-Test Plan 里面添加
–Foreach controller 组件生成的组件

CSV Data Set Config组件：
-更容易理解和使用
-适合大数据使用场景

如果recycle option 和stop Thread 都是false, 那么当到达文件尾部的时候，变量的数值被设置成,当然，这个可以通过修改属性文件中的csvdataset.eofstring 进行修改。

Jmeter中的属性

Jmeter 属性在测试脚本的任何地方都是可见的（全局）
Jmeter属性对于整个测试计划都是可见的（全局），因此可以用于在线程间传递信息。
属性是所有线程所共享的（全局）

配置jmeter.properties 可以应用属性到jmeter
Jmeter>

变量

可以在Test Plan添加
Jmeter中变量是局部的， 在不同测试线程中，可以是相同，也可以是不同的

Debug Sampler

不需要和网络进行交互的调试变量的方式
可以验证跟踪所有变量的值
基础配置组件中，user define variable不受作用域限制，无论身处哪个层级，里面的参数都能被所有的线程所拿到并使用

Logic Controller

Jmeter的controller是控制脚本的执行的顺序

  1. Simple Controller
    -提供一个独立的结构和控制，更方便，更清晰。
    -嵌套其他的controller
    -适用场景（连续请求处理）

  1. Loop Controller
    采样器放在Loop Controller的子域，则可以对该采样器起作用。

  2. Once only controller
    只执行一次的采样器

  3. For each controller
    这个组件中生成的值可以使用在其他线程中。一般是和User Defined Viriable 相绑定的。

    Generate an additional sample which measures the overall time taken to perform the nested test elements.
