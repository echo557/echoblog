左连接，两张表。 返回包括左表中的所有记录和右表中联结字段相等的记录

select * from users where id<(select max(id) from users) limit 1

Python 基础

1.什么是装饰器
装饰器是Python 函数， 它可以让其他函数在不需要做任何代码变动的前提下增加额外功能对象。比如说学哈如日志、性能测试、事务处理、缓存、权限校验等场景。

装饰器不能装饰静态方法
比如@logging装饰器可以在函数执行时额外输出日志，@cache装饰过的函数可以缓存计算结果等。

  1. 字典，元组，列表的差别
    list是处理一组有序项目的数据结构，即你可以在一个列表中存储一个序列的项目。可改变，可嵌套。
    list = [‘’,’’,’’]
    元组： 不可表，通过圆括号存储。 可以嵌套。
    zoo=(‘’,’’,’’)
    字典：键值对方式存储。键/值是没有顺序的。
    {‘’:’’,’’:’’,’’:’’}

BDD--Lettuce

对比框架

TDD: Test-Driven Development
TDD 原理是在开发功能代码之前，先编写单元测试用例代码，确定需要编写什么产品代码。TDD的基本思路是通过测试来驱动开发的进行，但测试驱动开发并不只是单纯的测试工作，而是把需求分析、设计和质量控制量化的过程。
TDD首先考虑使用需求（对象、功能、过程、接口等），主要是编写测试用例框架对功能的过程和接口进行设计，而测试框架可以持续进行验证。

ATDD： Acceptance Test Driven Development

BDD: Behavior Driven Development
主要从用户的需求出发，强调系统行为。包括验收测试和客户测试驱动等极限编程的实践，作为对测试驱动开发的回应。

Jdave(Java)
Beha(PHP)

Python Multiple Threading

多线程技术

  1. thread 提供了低级别的、原始的线程以及一个简单的锁
  2. threading 基于Java的线性涉及的. 锁（lock）和条件变量（condition）再JAVA 中是对象的基本行为（每一个对象都自带了锁和条件变量），而在Python中则是独立的对象。

我们应该避免thread 模块，因为它不支持守护线程。
threading支持守护线程。

例子

Example 1:
from time import sleep,ctime
import threading

def music(func,loop):
for i in range(loop):
print(“I was listen to %s! %s” %(func,ctime()))
sleep(2)

def movie(func,loop):
for i in range(loop):
print(“I was watching at %s! %s” %(func,ctime()))
sleep(2)

#create threads lists
threads =[]

#create Thread() by threading
t1 = threading.Thread(target=music, args=(‘Listen’,2))
threads.append(t1)

t2 = threading.Thread(target=movie, args=(‘watching good movie’,2))
threads.append(t2)

if name==”main“:

#start threads
for t in threads:
t.start()

#keep threads
for t in threads:
t.join()

print("all ends: %s" %ctime())

当然，我们还可以自定义线程
class MyThread(threading.Thread):
def init(self, func, args,name=’’):
threading.Thread.init(self)
self.func = func
self.args = args
self.name = name

def run(self):
    self.func(*self.args)

多线程技术multiprocessing

multiprocessing提供了本地和远程的并发性，有效地沟通全局解释锁(Global Interceptor Lock, GIL)来使用进程。
multiprocessing 可以利用硬件地多处理器来进行工作。支持Unix 和 Windows

修改上面地例子，将threading模块中地Thread方法替换为multiprocessing 模块地Process 即可

Pipe 和Queue

multiprocessing提供IPC（进程间通信），效率更高。
优先考虑Pipe和Queue，避免使用Lock/Event/Semaphore/Condition 等同步方式（因为他们占据的不是用户进程的资料）
