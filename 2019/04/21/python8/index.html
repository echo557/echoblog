Python Multiple Threading

多线程技术

  1. thread 提供了低级别的、原始的线程以及一个简单的锁
  2. threading 基于Java的线性涉及的. 锁（lock）和条件变量（condition）再JAVA 中是对象的基本行为（每一个对象都自带了锁和条件变量），而在Python中则是独立的对象。

我们应该避免thread 模块，因为它不支持守护线程。
threading支持守护线程。

例子

Example 1:
from time import sleep,ctime
import threading

def music(func,loop):
for i in range(loop):
print(“I was listen to %s! %s” %(func,ctime()))
sleep(2)

def movie(func,loop):
for i in range(loop):
print(“I was watching at %s! %s” %(func,ctime()))
sleep(2)

#create threads lists
threads =[]

#create Thread() by threading
t1 = threading.Thread(target=music, args=(‘Listen’,2))
threads.append(t1)

t2 = threading.Thread(target=movie, args=(‘watching good movie’,2))
threads.append(t2)

if name==”main“:

#start threads
for t in threads:
t.start()

#keep threads
for t in threads:
t.join()

print("all ends: %s" %ctime())

当然，我们还可以自定义线程
class MyThread(threading.Thread):
def init(self, func, args,name=’’):
threading.Thread.init(self)
self.func = func
self.args = args
self.name = name

def run(self):
    self.func(*self.args)

多线程技术multiprocessing

multiprocessing提供了本地和远程的并发性，有效地沟通全局解释锁(Global Interceptor Lock, GIL)来使用进程。
multiprocessing 可以利用硬件地多处理器来进行工作。支持Unix 和 Windows

修改上面地例子，将threading模块中地Thread方法替换为multiprocessing 模块地Process 即可

Pipe 和Queue

multiprocessing提供IPC（进程间通信），效率更高。
优先考虑Pipe和Queue，避免使用Lock/Event/Semaphore/Condition 等同步方式（因为他们占据的不是用户进程的资料）
