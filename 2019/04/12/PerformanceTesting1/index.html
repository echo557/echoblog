性能测试流程

测试准备

需求分析

  • 用户有明确需求
  • 用户无明确需求

建立性能目标

  • 选取核心业务流程（重要程度/频繁）

    • 登陆
    • 提交表单
    • Sync

  • 并发用户数（TPS）

  • 事务吞吐量需求
  • 响应时间需求
  • 系统占用资源需求
  • 可扩展性需求

了解应用软件

  • 系统体系架构
  • 采用的开发语言
  • 通信协议
  • 动态session
  • 验证码问题

了解软件部署平台

  • 物理部署（局域网/虚拟机）
  • 硬件架构（机型/CPU/MEM/IO/网络）
  • 操作系统（版本/关键内核参数）
  • 数据库（类型/版本/启动参数/存储布局）
  • 中间件（类型/线程数/内存参数）
  • 软件部署模式

建立系统负载模型

  • 业务层面

    • 核心业务流程吞吐量
    • 高峰期业务分布时段

  • 系统负载

    • 高峰/平均场景吞吐量
    • CPU/IO/MEM/NETWORK

  • 数据来源

    • 服务器端监控
    • 数据库日志
    • 用户提出需求

制定项目计划和方案

搭建环境

  • Eclipse
  • Jmeter
  • 各种监控工具

服务器环境

  • Linux 系统
  • 数据库

  • 各种监控工具

  • 使系统适应测试， 如去掉验证码

增强脚本

  • 参数化用户输入
  • 关键数据
  • 增加事务
  • 增加检查点

调试脚本

  • Vugen 单次回放
  • Vugen 多次回放
  • Controller 单脚本多用户
  • Controller 多脚本多用户
  • 查看日志回放

验证脚本

  • 通过Loadrunner检查点验证
  • 通过查看后台服务器日志验证
  • 通过测试系统查看运行后变化
  • 利用sql 查询/插入/更改效果
