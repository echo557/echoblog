Jmeter

Jmeter5

采样器

基于协议： HTTP, FTP, HTTPS 等

HTTP Request： 常用HTTPCLient4
常用设置： jmeter.properties
Workbench 中的mirror server 可以帮助调试HTTP 请求
去请求文件下载，使用的是POST请求。如果需要保存server所推过来的文件/response保存下来，则可以添加一个listner 去把文件存储下来： Save Response to a file， 下载之后jmeter 会将其保存在jmeter 的bin 目录下。

如果想要通过jmeter上传文件，则也可以upload， 但一般情况下，jmeter 上传文件后消耗会到网络端，而不会很好的测试到server 的性能。

默认解析器： htmlparser
通过htmlparser.classname
通过设置Retrive All Embedded Resources from HTML Files 和 Use Concurrent Pool 更真实的模拟负载

URLS must match 可以帮忙过滤某些特定的url

IP 欺骗> Source Address
作为IP欺骗的地址，所使用的IP 地址不能是动态的，必须是静态的。

File协议
可以读取到本地的文件，并读取出来。
加入后置处理器后，就可以进行使用。

参数化

那些需要参数化

登陆认证信息
一些和时间相关的，违反时间约束的
一些受其他字段约束的
一些来自于其他数据源（如数据库的）
其他在系统运行过程中需要变动的

变量是每个线程都有各自一份的，一个线程修改并不影响另一个线程。

实现方式：
-通过CSV Data Set Config 组件
-User Defined Variables
-Test Plan 里面添加
–Foreach controller 组件生成的组件

CSV Data Set Config组件：
-更容易理解和使用
-适合大数据使用场景

如果recycle option 和stop Thread 都是false, 那么当到达文件尾部的时候，变量的数值被设置成,当然，这个可以通过修改属性文件中的csvdataset.eofstring 进行修改。
