Version manage(Roll back & move forward)

  • HEAD

  • Method

    • git reset –hard xxxx
    • git reset –hard HEAD^
    • git reset –hard HEAD~n (roll back n steps)

  • reset commad

    • –soft(move pin in local workspace)
    • –mixed(move pin in local workspace, and reset temporary space)
    • –hard(move pin in local workspace, and reset temporary space and workspace)

Version manage(Reset after delete)

  • predcondition:before delete, files had updated to locl workspace

  • git reset –hard xxxx

    • delete action has already commited to local space, pin located to history record
    • delete action has not commited to local space, pin located to use HEAD

Branch

  • Advantage

    • Multiple branch development, prompt efficient
    • One branch failed has no effect to other braches

  • Command

    • Create: git branch branchname
    • Check : git branch -v
    • Switch: git checkout branchname

    • Merge:

      • Switch to target branch: git checkout target_branchname
      • Execute: git merge newcontent_branchname

  • Conflict

    • Step1: Edit file and delete special symbol
    • Step2: Edit file until it satisfy you ,quit file
    • git add filename

    • git commit -m “log information”

      • Note: commit can not include filename
