CORS 预请求

HTTP

HTTP-Protocol基础

URI,URL,URN

URI：Uniform Resource Identifier/统一资源标志符
-用来唯一标识互联网上的信息资源
-包括URL和URN

URL：Uniform Resource Locator/统一资源定位器
schema: ftp/http
路由：/path
此类格式都叫URL，比如ftp协议

URN：永久统一资源定位符
在资源移动之后还能被找到

报文

（a） 请求报文
–Method（如Get）+ 路由信息（/path）+ HTTP/1.0(HTTP版本)
–HTTP Header

（b）响应报文
– HTTP/1.0（版本）+status(200) + OK(Code的含义)
– HTTP Header
– HTTP Body（Body 和 Header之间有一个空行）

HTTP方法

用来定义对于资源的操作
常用： GET, POST, PUT, DELETE

HTTP code
–定义服务器对请求的处理结果
–各个区间的CODE有各自的语义
–好的HTTP服务可以通过CODE判断结果
1xx:通知
2xx: 成功
3xx：重定向
4xx：客户端错误
5xx: 服务端错误
200
401
404
500
503
