HTMLTestRunner

Download

http://tungwaiyip.info/software/HTMLTestRunner_0_8_2/HTMLTestRunner.py

注意事项

  1. HTMLTestRunner是基于Python2 开发的，在Python3上使用，需要修改某些代码
  2. 下载文件之后，需要把文件复制到Python的Lib目录下
  3. 使用方法：使用main函数进行调用
    suite = unittest.TestSuite()
    suite.addTest(TestMethod(“test_add1”))
    suite.addTest(TestMethod(“test_add2”))
    fp = open(“./result.html”,”wb”)
    runner = HTMLTestRunner(stream=fp, title=”result report”, description=u”Details”)
    runner.run(suite)
    fp.close()

Trouble Shooting

  1. Pycharm 中main函数不会执行的问题解决：
    Root Cause: 因为没有设置运行设置，目前可以看到默认运行的设置是：pytest。所以执行该文件就不会运行到main 函数
    1）修改执行configuration
    2）Edit
    3）有三个选择：
    –Python
    –Python tests
    –Templates
    4） 选择python, 点击添加，放入脚本路径，保存
    5）执行，就可生成

报告优化

  1. 增加注释
  2. 注释有两种： comment和doc string。 其中前者是普通注释，后者用于函数，类和方法的描述
    增加注释可以让报告阅读更容易
  3. 增加时间戳可以让旧报告不会被覆盖。方法：
    time.time() 获取当前时间戳
    time.ctime() 当前时间的字符串形式
    time.localtime() 当前时间的struct_time形式
    time.strftime() 用来获取当前时间，可以将时间格式化为字符串。
    比如：
    now=time.time.strftime(“%Y-%m-%d %H_%M_%S”)
    filename=’./‘+now+’result.html’

unitest

unittest的学习

assert() python所提供的方法， 第一份参数和第二进行比较，如果返回是true,则pass

unittest使用

test.py
import unittest
from comm.webCustomer.count import Count

class TestMethod(unittest.TestCase):
def setUp(self):
print(“test case start”)

def tearDown(self):
    print("test case end")

def test_add1(self):
    j = Count(2, 4)
    add = j.add()
    self.assertEqual(add, 7)

def test_add2(self):
    j = Count(2, 4)   
    add = j.add()
    self.assertEqual(add, 6)

if name==”main“:
unittest.main()

调用实现的时候，还可以使用以下方法：

suite = unittest.TestSuite()
suite.addTest(TestMethod("test_add1"))
suite.addTest(TestMethod("test_add2"))
runner = unittest.TextTestRunner()
runner.run(suite)

####　断言方法
assertEqual(a,b) 检查 a==b
assertNotEqual(a,b) 检查 a!=b
assertTrue(x) 检查bool(x) is True
assertFalse(x) 检查bool(x) is False
assertIs(a,b) a is b
assertIsNot(a,b) a is not b
assertIsNotNone(x) x is none
assertIn(a,b) a in b
assertNotIn(a,b) a not in b
assertIsInstance(a,b) isinstance(a,b)
assertNotIsInstance(a,b) not isinstance(a,b)

使用方法：
assertEqual(first,second,msg=None)
判断第一个参数和第二个参数是否相等，如果不相等则测试失败。 msg是可选参数，用于定义测试失败时打印的信息。

共同的测试类可以进行复用

有多个类就需要建不同的文件

多个suite一起测试

方法一：

  1. 新建一个文件py 文件，导入testsuite包
  2. 构建如下代码：
    import unittest
    from comm.webCustomer import webCustomer

suite = unittest.TestSuite()
suite.addTest(webCustomer.TestMethod(“test_add1”))
suite.addTest(webCustomer.TestMethod(“test_add2”))
if name==”main“:
runner = unittest.TextTestRunner()
runner.run(suite)

方法二：
使用discover的方法，discover 可以匹配到测试用例的测试套件
import unittest

testunit = unittest.TestSuite()
test_dir = ‘D:\code\AUT\comm\webCustomer’
discover = unittest.defaultTestLoader.discover(test_dir,
pattern=’test*.py’,
top_level_dir=None)

if name==”main“:
runner = unittest.TextTestRunner()
runner.run(discover)

测试用例归类

注意点：

  1. 需要以test 开头
  2. 可以以不同的测试功能来进行分类
  3. 可以参考功能测试的分类方法

discover方法找的是所给目录下的匹配条件
是按照ascii🐴的顺序来找测试用例的
unittest.main() 也是按照ascii码来文件的

跳过测试和预期失败

  1. unitest.skip(reason) 无条件跳过装饰的测试，说明跳过测试的原因
  2. unitest.skipIf(condition,reason) 条件为真，跳过
  3. unitest.skipUnless(condition,reason) 除非条件为真，跳过
  4. unitest.exceptedFailure() 测试标记失败，不管结果是否真的失败，标记为失败

  2. 类 setUpClass()/tearDownClass()
  3. 用例 setUp()/tearDown()
