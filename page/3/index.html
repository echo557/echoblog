对比框架

TDD: Test-Driven Development
TDD 原理是在开发功能代码之前，先编写单元测试用例代码，确定需要编写什么产品代码。TDD的基本思路是通过测试来驱动开发的进行，但测试驱动开发并不只是单纯的测试工作，而是把需求分析、设计和质量控制量化的过程。
TDD首先考虑使用需求（对象、功能、过程、接口等），主要是编写测试用例框架对功能的过程和接口进行设计，而测试框架可以持续进行验证。

ATDD： Acceptance Test Driven Development

BDD: Behavior Driven Development
主要从用户的需求出发，强调系统行为。包括验收测试和客户测试驱动等极限编程的实践，作为对测试驱动开发的回应。

不同语言下的BDD 框架

Cucumber(Ruby)
Jdave(Java)
Beha(PHP)
Behave(Python) http://pythonhosted.org/behave
Lettuce(Python) http://Lettuce.it

原理

通过装饰器来做base类，然后再写一些不一样的场景来实现测试用例的增加。

通过场景可以完成测试用例的编写

写行为的时候需要去匹配封装的那些行为。

Python Multiple Threading

多线程技术

  1. thread 提供了低级别的、原始的线程以及一个简单的锁
  2. threading 基于Java的线性涉及的. 锁（lock）和条件变量（condition）再JAVA 中是对象的基本行为（每一个对象都自带了锁和条件变量），而在Python中则是独立的对象。

我们应该避免thread 模块，因为它不支持守护线程。
threading支持守护线程。

例子

Example 1:
from time import sleep,ctime
import threading

def music(func,loop):
for i in range(loop):
print(“I was listen to %s! %s” %(func,ctime()))
sleep(2)

def movie(func,loop):
for i in range(loop):
print(“I was watching at %s! %s” %(func,ctime()))
sleep(2)

#create threads lists
threads =[]

#create Thread() by threading
t1 = threading.Thread(target=music, args=(‘Listen’,2))
threads.append(t1)

t2 = threading.Thread(target=movie, args=(‘watching good movie’,2))
threads.append(t2)

if name==”main“:

#start threads
for t in threads:
t.start()

#keep threads
for t in threads:
t.join()

print("all ends: %s" %ctime())

Example 2:
当线程很多的时候如何解决问题
from time import sleep,ctime
import threading

def super_player(file_,time):
for i in range(2):
print(“I was listen to %s! %s” %(file_,ctime()))
sleep(time)

lists={‘Test1’:3,’Test2’:5,’Test3’:6}

threads =[]
files = range(len(lists))
for file_,time in lists.items():
t = threading.Thread(target=super_player, args=(file_,time))
threads.append(t)

if name==”main“:
for t in threads:
t.start()

for t in threads:
    t.join()

当然，我们还可以自定义线程
class MyThread(threading.Thread):
def init(self, func, args,name=’’):
threading.Thread.init(self)
self.func = func
self.args = args
self.name = name

def run(self):
    self.func(*self.args)

多线程技术multiprocessing

multiprocessing提供了本地和远程的并发性，有效地沟通全局解释锁(Global Interceptor Lock, GIL)来使用进程。
multiprocessing 可以利用硬件地多处理器来进行工作。支持Unix 和 Windows

修改上面地例子，将threading模块中地Thread方法替换为multiprocessing 模块地Process 即可

Pipe 和Queue

multiprocessing提供IPC（进程间通信），效率更高。
优先考虑Pipe和Queue，避免使用Lock/Event/Semaphore/Condition 等同步方式（因为他们占据的不是用户进程的资料）

多线程测试可以用在性能测试中，可以结合性能测试的方向来进行思考

Selenium Grid2

工作原理

Grid 是用于设计进行分布式测试的工具。结构由一个hub 主节点和若干个node代理节点组成。 hub用来管理各个代理节点的注册和状态信息，并且接收远程客户端代码的请求调用，然后把请求的命令再转发给代理节点来执行。使用Grid远程执行测试的代码与直接调用Selenium Server 是一样的，知识环境启动的方式不一样，需要同时启动hub和至少一个node.

Download

https://www.seleniumhq.org/download/
Selenium Standalone Server

启动

java -jar selenium-server-standalone-3.141.59.jar -role hub
java -jar selenium-server-standalone-3.141.59.jar -role node(默认端口是4444)
启动多个节点：
java -jar selenium-server-standalone-3.141.59.jar -role node -port 5555
java -jar selenium-server-standalone-3.141.59.jar -role node -port 5556
java -jar selenium-server-standalone-3.141.59.jar -role node -port 5557

控制台地址：（指显示节点信息）
http://127.0.0.1:4444/grid/console

使用

单个浏览器调用
from selenium.webdriver import Remote

driver = Remote(command_executor=”http://127.0.0.1:4444/wd/hub",
desired_capabilities={‘platform’:’ANY’,
‘browserName’:’chrome’,
‘version’:’’,
‘javascriptEnabled’:True,

})

driver.get(“http://www.baidu.com")
driver.find_element_by_id(“kw”).send_keys(“remote”)
driver.find_element_by_id(“su”).click()
driver.quit()
print(‘end’)

多个节点上跑：

browser={‘http://127.0.0.1:4444/wd/hub':'chrome',

print(bro,hub)

desired_capabilities={‘platform’:’ANY’,
‘browserName’:bro,

                })

trouble shooting:
需要把相关的browser driver驱动放在python路径下

远程

如果要在其他主机上启动node,则需要满足以下条件：

  1. 本地hub主机与远程node主机之间可以ping通
  2. 远程主机需要安装用例执行的浏览器及驱动。并且驱动再环境变量的path的目录下
  3. 远程主机必须安装JAVA 环境，因为需要运行Selenium Server

步骤：

  1. 启动本地hub主机
  2. 启动远程node主机， 需要指向的hub主机IP
  3. 修改远程主机的IP地址及端口号，再其上面的firefox浏览下运行脚本

HtmlUnit 和PhantomJS 是没有界面浏览器， 所以其运行速度非常快。
